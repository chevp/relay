Quickstart

Up and running in 3 steps

1

Install

# From nuna monorepo
cd tools/relay
pnpm install
2

Start the server

# From any game repo root
pnpm exec relay serve

# Or open in player (combo: serve + nuna://play)
pnpm exec relay open
3

Verify

curl http://localhost:3001/discover.json
curl http://localhost:3001/health

CLI Reference

All flags and options

relay serve [options]
Flag Default Description
-p, --port <n> 3001 HTTP port to bind
-h, --host <addr> 0.0.0.0 Host address to bind
-c, --config <path> ./nuna-serve.xml Path to XML config file
-a, --assets <path> ./assets-export Exported assets directory (not source files)
--configs <path> ./configs Game configs directory (JSON/YAML)
--open false Launch nuna://play?discovery=... via OS handler
--watch false Regenerate manifests on file change (debounced 500ms)
--verbose false Enable HTTP request logging
--no-color — Disable ANSI color output
--version — Print version and exit
--help — Print usage and exit

Asset Pipeline

Clear separation between authoring, export, and serving

block assets-source/ NOT served

Blender (.blend), Photoshop (.psd) — stay local, never served.

inventory assets-export/ served at /assets/
.gltf
.png
.mat

Cache-Control: max-age=31536000 (immutable, hash-versioned)

tune configs/ served at /configs/
.json
.yaml

Cache-Control: no-cache (dynamic, live-editable)

warning

Authoring Format Detection

On startup, nuna-serve scans static roots for source files (.blend, .psd, .ai, .sketch) and emits a warning. These files belong in assets-source/, not in served paths.

XML Configuration

Declarative server config via nuna-serve.xml

<?xml version="1.0" encoding="UTF-8"?> <nuna-serve xmlns="https://nuna.dev/schemas/serve/v1" host="localhost" port="3001"> <!-- Statically hosted content roots --> <static> <root alias="game" path="." mount="/games/current/v-dev/"/> <root alias="kit" path="../nuna-kit" mount="/kit/v-dev/"/> <root alias="mods" path="./mods" mount="/mods/" optional="true"/> </static> <!-- Connected domain servers (advertised, not proxied) --> <servers> <server alias="game-server" url="ws://localhost:8080/ws" kind="gameplay" required="false"/> <server alias="persistence" url="http://localhost:8081" kind="save-load" required="false"/> </servers> <!-- Kit-source alias mapping --> <sources> <source alias="first-party" ref="static:kit"/> <source alias="game" ref="static:game"/> </sources> </nuna-serve>
folder

<static>

Mount filesystem directories as HTTP paths. Each root gets its own manifest.json.

alias — logical name

path — filesystem path

mount — HTTP prefix

optional — skip if missing

cloud

<servers>

Advertise connected domain servers. nuna-serve announces them, the renderer connects directly.

alias — logical name

url — http/ws endpoint

kind — role tag

required — ping on startup

link

<sources>

Map logical source aliases to static roots or servers. The renderer resolves these at runtime.

alias — source name

ref — static:<x> or server:<x>

info

No config? No problem.

If no nuna-serve.xml is found, nuna-serve auto-generates a minimal config: CWD as "game" root, assets-export/ as assets, and configs/ as game configs.

Discovery Endpoint

Single URL that describes the entire dev environment

GET /discover.json
{ "nunaServe": { "version": "0.1.0", "host": "localhost", "port": 3001 }, "static": { "game": { "url": "http://localhost:3001/games/current/v-dev/", "alias": "game" }, "kit": { "url": "http://localhost:3001/kit/v-dev/", "alias": "kit" } }, "servers": { "game-server": { "url": "ws://localhost:8080/ws", "kind": "gameplay", "required": true } }, "sources": { "first-party": "static:kit", "game": "static:game" } }
GET /discover.json

All mounts, servers, and source mappings in one document.

GET /health

Health check. Returns { status: "ok" }.

GET /<mount>/manifest.json

SHA-256 file manifest per static root. Auto-regenerated with --watch.

SHA-256 Manifests

Content-addressable file index for each static root

{ "version": 1, "generated": "2026-04-09T12:00:00Z", "files": { "worlds/main.world.xml": { "sha256": "a1b2c3d4e5f6...", "size": 1234 }, "models/hero.gltf": { "sha256": "f6e5d4c3b2a1...", "size": 56789 } } }
tag

Content Integrity

SHA-256 hash per file enables cache invalidation and integrity checks.

sync

Auto-Regeneration

With --watch, manifests are invalidated on file changes (500ms debounce).

Scope

What nuna-serve does and does not do

check_circle What it does

  • ■ Serve static game content over HTTP
  • ■ Generate SHA-256 file manifests
  • ■ Publish discovery endpoint for the renderer
  • ■ Advertise connected domain servers
  • ■ Watch files and regenerate manifests
  • ■ Warn about authoring files in served paths

cancel What it does not do

  • ■ Proxy domain server traffic
  • ■ Production deployment (HTTPS, auth, CDN)
  • ■ Asset build/optimization/FTP upload
  • ■ Blender/Photoshop export triggers
  • ■ Hot-reload in the renderer itself
  • ■ Binary packaging
