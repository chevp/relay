1. Prerequisites

RequirementVersionNotes
Node.js ≥ 20 Declared in package.json > engines.node. Check with node --version.
pnpm ≥ 8 The repo ships a pnpm-lock.yaml. Install with npm i -g pnpm if missing.
Git checkout — The package is not on npm (license: UNLICENSED). You install from this repo.
Why no npm install -g @nuna/cli? The package is private and not published to a registry. Install always happens from a local clone of nuna/nuna/tools/nuna-cli/.

2. Install (recommended: pnpm link)

This is the fastest workflow for a developer who also touches the CLI source. It puts nuna on your PATH as a symlink into the repo, so a rebuild is enough to pick up your changes — no reinstall.

1Install dependencies

cd c:/chevp/nuna/nuna/tools/nuna-cli
pnpm install

2Build the CLI

The bin entry points to ./dist/cli.js, so you need a TypeScript build before linking:

pnpm build

This runs tsc and produces dist/cli.js (and the rest of the compiled tree).

3Link the binary globally

pnpm link --global

That registers the package globally and creates a nuna shim on your PATH pointing at dist/cli.js.

Windows note. If nuna is not found after linking, your global pnpm bin folder is not on PATH. Run pnpm config get global-bin-dir and add the printed folder (typically %LOCALAPPDATA%\pnpm) to PATH, then open a new terminal.

3. Alternative install methods

3a. npm global install (no link)

Same outcome, no symlink — re-run on each rebuild:

cd c:/chevp/nuna/nuna/tools/nuna-cli
pnpm install
pnpm build
npm install -g .

3b. Run from source without installing

For one-off use, skip the build entirely and run TypeScript directly via tsx:

cd c:/chevp/nuna/nuna/tools/nuna-cli
pnpm install
pnpm dev -- serve --port 8080
# or
pnpm dev -- open ./games/my-game

The -- separates pnpm flags from the args forwarded to nuna.

3c. Run the built bundle directly

If you do not want it on PATH at all:

node c:/chevp/nuna/nuna/tools/nuna-cli/dist/cli.js serve

4. Verify the install

nuna --version
# 0.2.0

nuna --help
# Usage: nuna [options] [command]
#   serve     Start the Nuna dev HTTP server for the current game repo
#   open      Open a game in the Nuna player (combo: serve + nuna://play)
#   test      ...
#   build     ...
#   validate  ...
#   mcp       ...
#   story     ...

Smoke test in a game folder:

cd c:/chevp/nuna/nuna/games/my-game
nuna serve --port 8080

You should see a banner with the bind address, static roots, discovery URL (http://localhost:8080/discover.json) and health URL.

6. Updating & uninstalling

Update after a git pull

cd c:/chevp/nuna/nuna/tools/nuna-cli
pnpm install
pnpm build

If you used pnpm link --global, that is enough — the symlink picks up the new dist/. If you used npm install -g ., re-run that step.

Uninstall

# If installed via pnpm link --global
pnpm uninstall --global @nuna/cli

# If installed via npm install -g .
npm uninstall -g @nuna/cli

7. Troubleshooting

nuna: command not found after linking

The pnpm global bin folder is not on PATH.

pnpm config get global-bin-dir

Add the printed folder to PATH (Windows: System Properties → Environment Variables; macOS/Linux: append to ~/.zshrc or ~/.bashrc). Open a new terminal.

Cannot find module '.../dist/cli.js'

You linked before building. Run pnpm build in tools/nuna-cli/.

Permission errors on npm install -g . (macOS/Linux)

Either prefix with sudo, or — better — use a Node-version manager (nvm, fnm, volta) so the global prefix lives in your home directory.

Port already in use

nuna serve --port 8081

It opens a browser instead of nuna-player.exe

The nuna:// protocol handler is registered by the nuna-player Windows installer (nuna-player-setup.exe). Without it the OS has no handler and falls back to a browser. Install the player first, then retry nuna open.
